# Revops Forecasting

Use this skill when the work is about revenue predictability rather than copywriting or one-off deal support. It is optimized for stage hygiene, weighted pipeline logic, forecast reasoning, and the operating cadences that keep sales, marketing, customer success, and finance aligned.

## When to Use

- Running a weekly forecast call or pipeline review
- Redefining stage exit criteria or lead routing rules
- Explaining why conversion, velocity, or win rate changed
- Reconciling top-down plan, bottom-up forecast, and capacity
- Auditing CRM hygiene, close-date quality, or source attribution
- Designing renewal, expansion, or territory inspection cadences

Do not use this skill for brand messaging, legal drafting, or implementation onboarding. Those belong to other skills.

## Workflow

1. Normalize the operating lens.
   Decide whether the task is about new business, expansion, renewal, or blended revenue.
2. Check data trust first.
   Validate stage discipline, amount currency, close-date realism, owner completeness, and source consistency before making recommendations.
3. Diagnose funnel shape.
   Inspect conversion by stage, aging, slip rate, deal concentration, and pipeline coverage against quota.
4. Separate signal from noise.
   Call out one-time deals, channel mix shifts, seasonal effects, pricing changes, and campaign bursts explicitly.
5. Produce an operating decision.
   Recommend stage changes, qualification rules, routing changes, capacity shifts, or forecast adjustments with a clear owner and next review date.

## Output Shape

- Forecast summary: commit, best case, and risk-adjusted view
- Funnel diagnosis: stage, issue, evidence, likely root cause
- Data hygiene findings: field, impact, required cleanup
- Action plan: owner, action, expected effect, review date
- Assumptions: explicit and falsifiable

## Heuristics

- Prefer inspection metrics over vanity totals.
- Separate pipeline creation problems from stage-conversion problems.
- Never bury forecast assumptions. List them plainly.
- Renewal and expansion forecasts should include health signals, not only booked amount.
- If the CRM is untrustworthy, recommend cleanup before precision math.

## Anti-Patterns

- Using raw pipeline amount as forecast truth
- Hiding poor stage hygiene behind weighted formulas
- Mixing net-new, renewal, and expansion logic into one number
- Recommending territory changes without capacity or coverage evidence
